General Information

Authentication 🔐

  • Uses **JWT** via cookies (accessToken, refreshToken).
  • **Protected endpoints** require a valid access token via cookie.

Conventions 📜

  • **Content-Type**: application/json unless otherwise noted (multipart/form-data for file uploads).
  • **Standard Response Envelope**: { statusCode, data, message }.
  • **Pagination/Filtering**: Available on listing endpoints via query string where supported.

Static Assets 🖼️

  • GET /public/* → static files served from server/public
  • GET /uploads/* → uploaded assets (e.g., product images)

Auth Routes

Route Prefix: /api/v1/auth

GET /me(Protected)

Returns the current authenticated user.

POST /login(Public)

Step 1 of login: sends OTP code to email.

Body: { email: string, password: string }

POST /verify-code(Public)

Step 2 of login: verifies code and issues cookies. Also returns { accessToken, user }.

Body: { email: string, code: string }

POST /refresh-token(Protected)

Uses refreshToken cookie to rotate and return a new access token.

POST /logout(Protected)

Clears accessToken and refreshToken cookies.

Products Routes

Route Prefix: /api/v1/products

GET /(Public)

Returns paginated list of products. Supports productService list filters (e.g., pagination, search) via Query.

GET /slug/:slug(Public)

Returns product by slug.

GET /:id(Public)

Returns product by id.

POST /(Protected)

Creates a product. Uses multipart/form-data.

Fields:

  • images: file[] (max 10)
  • digitalFile: file (optional)
  • Other product fields in body (e.g., name, regular_price, is_digital_product)
PUT /:id(Protected)

Updates a product. Uses multipart/form-data. Can add new media; existing media_urls can be merged by providing them in the body.

DELETE /:id(Protected)

Deletes a product and attempts to remove stored images/files.

POST /bulk-delete(Protected)

Deletes multiple products.

Body: { ids: string[] }

PATCH /:id/toggle-publish(Protected)

Toggles product publish status.

Attributes Routes

Route Prefix: /api/v1/attributes

GET /(Protected)

Lists attributes.

POST /(Protected)

Creates an attribute.

PUT /:id(Protected)

Updates an attribute.

DELETE /:id(Protected)

Deletes an attribute.

PATCH /:id/toggle-status(Protected)

Toggles attribute status.

Variants Routes

Route Prefix: /api/v1/variants

GET /product/:productId(Protected)

Lists variants for a specific product.

POST /(Protected)

Creates a variant. Uses multipart/form-data (for images).

Categories Routes

Route Prefix: /api/v1/categories

GET /(Public)

Returns all categories.

POST /(Protected)

Creates a category. Uses multipart/form-data (for a single image).

PATCH /toggle-multiple-published(Protected)

Toggles publish status for multiple categories. Body: { ids: string[], action: 'publish' | 'unpublish' }

POST /import(Protected)

Imports categories. Uses multipart/form-data for a CSV or JSON file.

Collections Routes

Route Prefix: /api/v1/collections

GET /with-products(Public)

Returns collections with related products.

GET /:slug/products(Public)

Returns products belonging to the collection slug.

PATCH /toggle-multiple-featured(Protected)

Toggles featured status for multiple collections. Body: { ids: string[], action: 'feature' | 'unfeature' }

Inventory Routes

Route Prefix: /api/v1/inventory

POST /update-stock(Protected)

Updates stock quantity.

Body: { productId, variantId?, quantityDelta }

GET /:productId/:variantId(Protected)

Get inventory for a specific variant.

GET /:productId(Protected)

Get inventory for a product (all variants).

Orders Routes

Route Prefix: /api/v1/orders

CRUD operations are defined but **Authentication Status is Unknown**.

POST /(Protected/Unknown)

Create order.

Coupons Routes

Route Prefix: /api/v1/coupons

POST /apply(Protected/Unknown)

Apply a coupon to a cart/order context.

Developer Notes 📝

  • Endpoints marked **Protected/Unknown** may require authentication based on business requirements, even if not enforced in the route file. Verify in controllers/services.
  • File upload fields use a storage layer writing into /uploads. Returned URLs are accessible under the static route /uploads.
  • **CORS** is restricted by the ALLOWED_ORIGINS environment variable.
